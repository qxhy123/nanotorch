## Overview

YOLO v2 (YOLO9000) was developed by Joseph Redmon and Ali Farhadi, being the second version of the YOLO series. It significantly improved detection accuracy while maintaining real-time performance.

### Main Features of YOLO v2

1. **Darknet-19 Backbone**: 19-layer convolutional network, faster than VGG-16
2. **Batch Normalization**: Applied to all convolutional layers
3. **Anchor Boxes**: K-means clustering for anchor generation
4. **Direct Location Prediction**: Uses sigmoid for relative offset prediction
5. **Passthrough Layer**: Fine-grained feature fusion
6. **Multi-scale Training**: Supports different input sizes
7. **High-Resolution Classifier**: First train at 224×224, then fine-tune at 448×448

## YOLO v2 Core Improvements

### 1. Darknet-19 Backbone Network

Darknet-19 is a 19-layer convolutional network:

```
Input (3×416×416)
    ↓ Conv 3×3 (32) → MaxPool
    ↓ Conv 3×3 (64) → MaxPool
    ↓ Conv 3×3 (128) → Conv 1×1 (64) → Conv 3×3 (128) → MaxPool
    ↓ Conv 3×3 (256) → Conv 1×1 (128) → Conv 3×3 (256) → MaxPool
    ↓ Conv 3×3 (512) → [Conv 1×1 (256) → Conv 3×3 (512)] ×4 → MaxPool
    ↓ Conv 3×3 (1024) → [Conv 1×1 (512) → Conv 3×3 (1024)] ×2
    ↓ Output (1024×13×13)
```

Improvements over YOLO v1:
- **Fewer parameters**: 5.5M vs 40.2M
- **Faster speed**: Higher efficiency
- **Better accuracy**: Regularization effect from BN

### 2. Batch Normalization

YOLO v2 applies batch normalization to all convolutional layers:
- **Regularization effect**: Reduces overfitting
- **Faster convergence**: Quicker training
- **No Dropout needed**: BN is sufficient

### 3. Anchor Boxes and K-means Clustering

YOLO v2 uses K-means clustering to automatically discover optimal anchor boxes:

```python
# 5 anchor boxes for VOC dataset
VOC_ANCHORS = [
    (1.3221, 1.73145),   # Small objects
    (3.19275, 4.00944),  # Medium objects
    (5.05587, 8.09892),  # Large objects
    (9.47112, 4.84053),  # Wide objects
    (11.2364, 10.0071)   # Extra large objects
]
```

### 4. Direct Location Prediction

YOLO v2 uses sigmoid for location prediction:

```
bx = σ(tx) + cx
by = σ(ty) + cy
bw = pw * exp(tw)
bh = ph * exp(th)
```

Where:
- `(cx, cy)`: Grid cell's top-left coordinates
- `(pw, ph)`: Anchor box dimensions
- `(tx, ty, tw, th)`: Network outputs

### 5. Passthrough Layer

Passthrough layer fuses high-resolution features with low-resolution features:

```
High-resolution feature (26×26×512)
        ↓ Passthrough (stride=2)
        ↓ Reshape to (13×13×2048)
        ↓ Conv 1×1 → (13×13×256)
        ↓ Concat with main feature
        ↓ Final feature
```

## Model Architecture

### 1. ConvBN (Basic Convolution Block)

```python
from nanotorch.detection.yolo_v2 import ConvBN
from nanotorch.tensor import Tensor
import numpy as np

conv_bn = ConvBN(64, 128, kernel_size=3, stride=1, padding=1)
x = Tensor(np.random.randn(1, 64, 52, 52).astype(np.float32))
y = conv_bn(x)
print(y.shape)  # (1, 128, 52, 52)
```

### 2. Darknet19 Backbone Network

```python
from nanotorch.detection.yolo_v2 import Darknet19

backbone = Darknet19(in_channels=3)
x = Tensor(np.random.randn(1, 3, 416, 416).astype(np.float32))
features = backbone(x)

print(features['output'].shape)   # (1, 1024, 13, 13)
print(features['route1'].shape)   # (1, 256, 52, 52)
print(features['route2'].shape)   # (1, 512, 26, 26)
```

### 3. Passthrough Layer

```python
from nanotorch.detection.yolo_v2 import PassthroughLayer

passthrough = PassthroughLayer(stride=2)
x = Tensor(np.random.randn(1, 512, 26, 26).astype(np.float32))
y = passthrough(x)
print(y.shape)  # (1, 2048, 13, 13)
```

### 4. Complete YOLOv2 Model

```python
from nanotorch.detection.yolo_v2 import YOLOv2, YOLOv2Tiny, build_yolov2

# Method 1: Create directly
model = YOLOv2(num_classes=20, input_size=416)

# Method 2: Create Tiny version
model = YOLOv2Tiny(num_classes=20, input_size=416)

# Method 3: Use factory function
model = build_yolov2('full', num_classes=20, input_size=416)
model = build_yolov2('tiny', num_classes=20, input_size=224)

# Forward pass
x = Tensor(np.random.randn(1, 3, 416, 416).astype(np.float32))
output = model(x)

# Output shape: (1, 125, 13, 13)
# 125 = 5 anchors × (4 bbox + 1 conf + 20 classes)
print(output['output'].shape)
```

## Loss Function

### YOLO v2 Loss Function Design

YOLO v2 uses a multi-part loss function:

$$
L = \lambda_{coord} L_{coord} + \lambda_{obj} L_{obj} + \lambda_{noobj} L_{noobj} + \lambda_{class} L_{class}
$$

**Coordinate Loss** (using MSE):

$$
L_{coord} = \sum_{i=0}^{S^2} \sum_{j=0}^{B} \mathbb{1}_{ij}^{obj} \left[ (t_x - \hat{t}_x)^2 + (t_y - \hat{t}_y)^2 + (t_w - \hat{t}_w)^2 + (t_h - \hat{t}_h)^2 \right]
$$

**Confidence Loss** (using BCE):

$$
L_{obj} = -\sum_{i,j} \mathbb{1}_{ij}^{obj} \left[ C_i \log(\hat{C}_i) + (1 - C_i) \log(1 - \hat{C}_i) \right]
$$

$$
L_{noobj} = -\lambda_{noobj} \sum_{i,j} \mathbb{1}_{ij}^{noobj} \left[ C_i \log(\hat{C}_i) + (1 - C_i) \log(1 - \hat{C}_i) \right]
$$

**Classification Loss** (using Softmax cross-entropy):

$$
L_{class} = -\sum_{i=0}^{S^2} \mathbb{1}_{i}^{obj} \sum_{c=1}^{C} y_c \log(\hat{p}_c)
$$

### Anchor K-means Clustering

YOLO v2 uses K-means clustering to automatically determine anchor sizes:

$$
d(\text{box}, \text{centroid}) = 1 - \text{IoU}(\text{box}, \text{centroid})
$$

### Passthrough Layer

Passthrough layer reorganizes high-resolution features:

$$
\text{Output}_{26 \times 26 \times 512} \rightarrow \text{Reshape} \rightarrow \text{Output}_{13 \times 13 \times 2048}
$$

$$
\text{Final} = \text{Concat}(\text{Passthrough}, \text{Conv}_{13 \times 13}) \in \mathbb{R}^{13 \times 13 \times 3072}
$$

### Using the Loss Function

```python
from nanotorch.detection.yolo_v2 import YOLOv2Loss

loss_fn = YOLOv2Loss(
    num_classes=20,
    lambda_coord=5.0,     # Coordinate loss weight
    lambda_obj=1.0,       # Object confidence loss weight
    lambda_noobj=0.5,     # No-object confidence loss weight
    lambda_class=1.0,     # Classification loss weight
    ignore_threshold=0.6  # IoU ignore threshold
)

# Compute loss
predictions = {
    'output': Tensor(np.random.randn(2, 125, 13, 13).astype(np.float32) * 0.1)
}

targets = [
    {'boxes': np.array([[100, 100, 200, 200]], dtype=np.float32),
     'labels': np.array([0], dtype=np.int64)},
]

loss, loss_dict = loss_fn(predictions, targets)
print(f"Total Loss: {loss.item():.4f}")
print(f"Coord Loss: {loss_dict['coord_loss']:.4f}")
print(f"Obj Loss: {loss_dict['obj_loss']:.4f}")
print(f"NoObj Loss: {loss_dict['noobj_loss']:.4f}")
print(f"Class Loss: {loss_dict['class_loss']:.4f}")
```

## Training Process

### Complete Training Example

```python
from nanotorch.tensor import Tensor
from nanotorch.optim import Adam
from nanotorch.optim.lr_scheduler import StepLR
from nanotorch.detection.yolo_v2 import build_yolov2, YOLOv2Loss
import numpy as np

# Create model
model = build_yolov2('tiny', num_classes=20, input_size=224)

# Loss function and optimizer
loss_fn = YOLOv2Loss(num_classes=20)
optimizer = Adam(model.parameters(), lr=1e-4)
scheduler = StepLR(optimizer, step_size=10, gamma=0.1)

# Training loop
for epoch in range(10):
    model.train()
    total_loss = 0
    
    for batch_idx in range(10):
        # Generate random data
        images = Tensor(np.random.randn(2, 3, 224, 224).astype(np.float32))
        targets = [{
            'boxes': np.random.rand(2, 4).astype(np.float32) * 200,
            'labels': np.random.randint(0, 20, 2).astype(np.int64)
        } for _ in range(2)]
        
        optimizer.zero_grad()
        output = model(images)
        
        # Compute loss
        loss, loss_dict = loss_fn(output, targets)
        
        # Backward pass
        loss.backward()
        optimizer.step()
        
        total_loss += loss.item()
    
    scheduler.step()
    avg_loss = total_loss / 10
    print(f"Epoch {epoch+1}, Loss: {avg_loss:.4f}")
```

## Common Issues

### 1. Why use K-means clustering for anchor boxes?

Traditional methods use hand-designed anchor box sizes, while K-means clustering:
- Automatically discovers optimal anchor sizes in data
- Better adapts to different datasets
- Improves detection accuracy

### 2. What is the purpose of Passthrough layer?

Passthrough layer:
- Passes high-resolution features to detection layer
- Improves small object detection
- Similar to skip connections in ResNet

### 3. Advantages of direct location prediction?

- **Numerical stability**: Sigmoid output in [0, 1] range
- **Physical meaning**: Predicts offset relative to grid
- **Faster convergence**: Avoids unstable position predictions

### 4. How to choose between Full vs Tiny versions?

| Version | Parameters | Speed | Accuracy | Use Case |
|---------|------------|-------|----------|----------|
| Full | 5.5M | Medium | High | Server/Desktop |
| Tiny | Fewer | Fast | Medium | Edge devices |
